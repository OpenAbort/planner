# UI Rules

When creating UI:

- Create feature-specific components under features/<feature>/components
- Prefer composition over large components
- Should one component per file
- Prefer hook over inline logic into component

When adding a new feature:

- Review existing features before creating new patterns.
- Reuse existing patterns when appropriate.
- If a new pattern would be significantly better than the current documented approach, explain the tradeoffs and ask for
  permission before changing the project conventions.
- If project conventions change, update this instruction file.
